## 摘要
云渲染广泛应用于游戏和 XR 领域，以克服客户端 GPU 资源有限的问题并支持异构设备。现有系统通常将渲染后的场景作为 2D 视频流交付，这使得传输内容与服务器渲染的视点紧密耦合，并将延迟补偿限制在图像空间重投影或扭曲（warping）上。在本文中，我们研究了一种替代方法，即流式传输实时的 3D 高斯泼溅（3D Gaussian Splatting, 3DGS）场景表示，而非仅传输渲染后的视频。我们展示了一个基于 Unity 的原型系统，其中服务器根据实时渲染的参考视图构建并持续优化 3DGS 模型，同时通过完整模型快照和支持重光照及刚体动力学的增量更新，将不断演化的表示流式传输给远程客户端。客户端在本地重建流式传输的高斯模型，并根据接收到的表示渲染其当前视点。与针对每个用户的渲染和视频流相比，该方法旨在提高延迟补偿的视点灵活性，并更好地在多个用户之间分摊服务器端的场景建模开销。我们描述了系统设计，对其进行了评估，并将其与传统的图像扭曲技术进行了比较。

## Abstract
Cloud rendering is widely used in gaming and XR to overcome limited client-side GPU resources and to support heterogeneous devices. Existing systems typically deliver the rendered scene as a 2D video stream, which tightly couples the transmitted content to the server-rendered viewpoint and limits latency compensation to image-space reprojection or warping. In this paper, we investigate an alternative approach based on streaming a live 3D Gaussian Splatting (3DGS) scene representation instead of only rendered video. We present a Unity-based prototype in which a server constructs and continuously optimizes a 3DGS model from real-time rendered reference views, while streaming the evolving representation to remote clients using full model snapshots and incremental updates supporting relighting and rigid object dynamics. The clients reconstruct the streamed Gaussian model locally and render their current viewpoint from the received representation. This approach aims to improve viewpoint flexibility for latency compensation and to better amortize server-side scene modeling across multiple users than per-user rendering and video streaming. We describe the system design, evaluate it, and compare it with conventional image warping.

### 1. 核心问题与研究动机

*   **研究动机**：利用 **3D 高斯泼溅（3DGS）** 这种显式场景表示，将其作为云渲染的中间层。通过流式传输不断演化的 3D 模型而非单纯的像素，使客户端能够自主进行视角合成，从而实现更灵活的延迟补偿和多用户场景下的服务器开销分摊。

### 2. 方法论
*   **核心思想**：在服务器端从 Unity 引擎实时提取几何与材质数据，在线构建并持续优化一个 3DGS 模型，同时将该模型的快照和增量更新流式传输至客户端。
*   **关键技术细节**：
    *   **在线构建与优化**：服务器管理“输入相机”（低分辨率，用于初始化高斯）和“参考相机”（高分辨率，用于微分渲染优化）。通过 Unity 渲染的 G-Buffer（位置、法线、颜色、物体 ID）直接生成高斯原语。
    *   **渐进式空间扩展**：将场景划分为 3D 网格，仅在用户探索到的新区域初始化高斯。引入**高斯冻结（Freezing）**机制，将稳定区域的高斯转为只读状态，以降低优化器的计算负担。
    *   **动态场景支持**：
        *   **刚体动力学**：通过物体 ID 将高斯与 Unity 中的游戏对象关联，仅传输变换矩阵（Transform）即可实现高斯群体的同步移动。
        *   **实时重光照**：利用球谐函数（SH）存储漫反射信息，并结合实时生成的阴影贴图（Shadow Map）估算每个高斯的可见性，支持光源方向变化。
    *   **流式传输协议**：采用混合模式。使用修改后的 **Draco 压缩**发送全量模型快照，在快照之间发送 **Tensor Deltas（张量增量）**，对位置、缩放、透明度等属性进行差分编码和 Zstd 压缩。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了固定相机收敛性测试、移动相机探索测试、动态物体跟踪测试以及流式传输带宽压力测试。
*   **充分性评价**：实验设计较为全面，不仅验证了视觉质量（SSIM/FLIP），还深入分析了流式传输的比特率波动（Bitrate Breakdown）。虽然测试场景的几何复杂度（如植被、透明物体）相对简单，但对于验证“在线构建+流式传输”这一原型系统而言，实验是客观且充分的。

### 6. 主要结论与发现
*   **视角灵活性**：3DGS 在处理视角偏移时的表现显著优于传统的图像扭曲技术，尤其是在处理遮挡关系时。
*   **收敛特性**：图像质量是“历史相关”的。一个区域被观察和优化的时间越长，渲染质量越高。
*   **带宽效率**：流式传输的平均比特率约为 92.5 Mbps（中位数 66.8 Mbps），虽然在发送全量快照时存在峰值（最高 481 Mbps），但增量更新能有效维持平稳传输。
*   **动态支持有效性**：显式处理物体变换和重光照能极大避免动态场景下的模糊现象。

### 7. 优点
*   **架构创新**：成功将 3DGS 从“离线重建工具”转化为“实时渲染传输媒介”，打通了游戏引擎到神经渲染的闭环。
*   **延迟补偿能力强**：客户端拥有完整的 3D 表示，可以实现真正意义上的本地 6-DOF（六自由度）视角合成。
*   **扩展性好**：网格划分和高斯冻结机制为处理大规模场景提供了可行路径。

### 8. 不足与局限
*   **质量收敛延迟**：用户进入新区域时，最初看到的图像质量较低，需要数秒优化才能达到高品质。
*   **材质表达有限**：目前的重光照模型主要针对漫反射，难以处理强镜面反射或复杂的全球照明效果。
*   **形变限制**：仅支持刚体变换，无法处理非刚体变形（如角色动画、流体）。
*   **带宽峰值**：Draco 快照导致的瞬时带宽需求极高，可能对网络抖动敏感。
