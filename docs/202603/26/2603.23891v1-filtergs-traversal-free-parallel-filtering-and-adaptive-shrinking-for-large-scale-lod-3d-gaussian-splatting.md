## 摘要
3D 高斯泼溅（3D Gaussian Splatting）凭借其实时性能彻底改变了神经渲染领域。然而，利用细节层次（LoD）方法将该技术扩展到大规模场景面临着严峻挑战：低效的串行遍历消耗了超过 60% 的渲染时间，且冗余的高斯-瓦片（Gaussian-tile）对产生了不必要的处理开销。为了解决这些局限性，我们提出了 FilterGS，其核心是一个包含两个互补过滤器的并行过滤机制，能够在无需树遍历的情况下高效选择高斯元素。此外，我们提出了一种全新的 GTC 指标，用于量化高斯-瓦片键值对的冗余度。基于该指标，我们引入了一种场景自适应的高斯收缩策略，有效减少了冗余对。大量实验表明，FilterGS 在多个大规模数据集上实现了最先进的渲染速度，同时保持了具有竞争力的视觉质量。项目主页：https://github.com/xenon-w/FilterGS

## Abstract
3D Gaussian Splatting has revolutionized neural rendering with real-time performance. However, scaling this approach to large scenes using Level-of-Detail methods faces critical challenges: inefficient serial traversal consuming over 60\% of rendering time, and redundant Gaussian-tile pairs that incur unnecessary processing overhead. To address these limitations, we introduce FilterGS, featuring a parallel filtering mechanism with two complementary filters that select Gaussian elements efficiently without tree traversal. Additionally, we propose a novel GTC metric that quantifies the redundancy of Gaussian-tile key-value pairs. Based on this metric, we introduce a scene-adaptive Gaussian shrinking strategy that effectively reduces redundant pairs. Extensive experiments demonstrate that FilterGS achieves state-of-the-art rendering speeds while maintaining competitive visual quality across multiple large-scale datasets. *   **核心问题**：在大规模场景中，3D 高斯泼溅（3DGS）为了平衡细节与效率通常采用细节层次（LoD）树结构。然而，现有 LoD 方法存在两大瓶颈：

    2.  **冗余计算开销**：生成了大量对最终图像贡献微乎其微的“高斯-瓦片（Gaussian-tile）”键值对，增加了排序和混合的负担。

### 2. 方法论
FilterGS 提出了两个核心创新模块：
*   **免遍历并行过滤机制（Parallel Filtering）**：
    *   **R&L 过滤器（半径与叶子）**：同时计算所有高斯的 2D 投影半径，保留半径小于阈值（足够精细）的节点或叶子节点。
    *   **祖先过滤器（Ancestor Filter）**：利用预计算的祖先路径，如果一个节点的祖先已经满足过滤条件，则剔除该节点，确保每个分支只保留一个最合适的层级。
    *   **并行化实现**：将过滤逻辑集成在单个 CUDA 内核中，使过滤耗时与树深度解耦，仅取决于高斯总数。
*   **场景自适应高斯收缩策略（Adaptive Shrinking）**：
    *   **KPC 指标**：定义“键值对贡献度”，量化单个高斯对特定瓦片（Tile）的有效像素贡献。
    *   **GTC 指标（高斯-瓦片贡献度）**：计算全场景/视角的平均贡献度，用于衡量场景的“拥挤”程度。
    *   **自适应收缩**：根据 GTC 动态调整剪枝阈值 $\tau$。在拥挤场景中采取更激进的收缩，显著减少冗余键值对，从而加速后续的排序和混合阶段。
